How this model works

The estimator divides tasks into four tiers based on complexity, each mapped to the most appropriate model for that tool. The Opus planning percentage determines what share of tasks use a frontier model; the remaining tasks are split proportionally across simple, medium, and complex tiers.

  • Simple (15k input / 1k output tokens) — quick edits, questions. Claude Code uses Haiku; Copilot uses GPT-5 mini; Cursor uses Auto mode.
  • Medium (35k / 3k) — bug fixes, feature additions. Claude Code uses Sonnet; Copilot uses GPT-4.1; Cursor uses Auto mode.
  • Complex (60k / 8k) — refactors, architecture. All three use Sonnet (or Max Sonnet for Cursor).
  • Planning / Opus (80k / 10k) — design sessions, requirements. Claude Code and Cursor pay full Opus API rates ($15/$75 per million tokens). Copilot's Opus rate is roughly one-third of that ($5/$25), reflecting its wholesale agreement with Anthropic.

Claude Code costs include prompt caching — repeated file reads within a session are billed at approximately 10% of the standard input rate. The cache hit rate slider controls how much of a reduction this produces. Cursor's $400/month floor (for 10 engineers) reflects the $20 platform fee per seat on top of the $20 in included credits.

These are estimates. Real costs depend on actual context sizes, session lengths, model routing decisions, and whether your team's usage stays within included credit pools. The model is intentionally transparent so you can adjust assumptions to match your team.
